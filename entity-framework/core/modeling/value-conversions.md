# <a name="value-conversions"></a><span data-ttu-id="b5f4b-102">值转换</span><span class="sxs-lookup"><span data-stu-id="b5f4b-102">Value Conversions</span></span>

> [!NOTE]  
> <span data-ttu-id="b5f4b-103">此功能是 EF Core 2.1 中的新增功能。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-103">This feature is new in EF Core 2.1.</span></span>

<span data-ttu-id="b5f4b-104">值转换器允许在读取或写入数据库时转换属性值。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-104">Value converters allow property values to be converted when reading from or writing to the database.</span></span> <span data-ttu-id="b5f4b-105">此转换可以是同一类型的另一个值（例如，加密字符串）或从一种类型的值转换为另一种类型的值（例如，在数据库中将枚举值与字符串相互转换）。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-105">This conversion can be from one value to another of the same type (for example, encrypting strings) or from a value of one type to a value of another type (for example, converting enum values to and from strings in the database.)</span></span>

## <a name="fundamentals"></a><span data-ttu-id="b5f4b-106">基础</span><span class="sxs-lookup"><span data-stu-id="b5f4b-106">Fundamentals</span></span>

<span data-ttu-id="b5f4b-107">值转换器根据 `ModelClrType` 和 `ProviderClrType`来指定。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-107">Value converters are specified in terms of a `ModelClrType` and a `ProviderClrType`.</span></span> <span data-ttu-id="b5f4b-108">模型类型是实体类型中的属性的 .NET 类型。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-108">The model type is the .NET type of the property in the entity type.</span></span> <span data-ttu-id="b5f4b-109">提供程序类型是数据库提供程序理解的 .NET 类型。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-109">The provider type is the .NET type understood by the database provider.</span></span> <span data-ttu-id="b5f4b-110">例如，若要将枚举作为字符串保存在数据库中，模型类型是枚举的类型，提供程序类型为 `String`。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-110">For example, to save enums as strings in the database, the model type is the type of the enum, and the provider type is `String`.</span></span> <span data-ttu-id="b5f4b-111">这两种类型可以相同。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-111">These two types can be the same.</span></span>

<span data-ttu-id="b5f4b-112">使用两个 `Func` 表达式树来定义转换：一个从 `ModelClrType` 到 `ProviderClrType`，另一个从 `ProviderClrType` 到 `ModelClrType`。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-112">Conversions are defined using two `Func` expression trees: one from `ModelClrType` to `ProviderClrType` and the other from `ProviderClrType` to `ModelClrType`.</span></span> <span data-ttu-id="b5f4b-113">使用表达式树，以便可以将它们编译到数据库访问代码中以便进行有效的转换。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-113">Expression trees are used so that they can be compiled into the database access code for efficient conversions.</span></span> <span data-ttu-id="b5f4b-114">对于复杂转换，表达式树可能是对执行转换的方法的简单调用。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-114">For complex conversions, the expression tree may be a simple call to a method that performs the conversion.</span></span>

## <a name="configuring-a-value-converter"></a><span data-ttu-id="b5f4b-115">配置值转换器</span><span class="sxs-lookup"><span data-stu-id="b5f4b-115">Configuring a value converter</span></span>

<span data-ttu-id="b5f4b-116">值转换是在 `DbContext`的 `OnModelCreating` 中的属性上定义的。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-116">Value conversions are defined on properties in the `OnModelCreating` of your `DbContext`.</span></span> <span data-ttu-id="b5f4b-117">例如，假设枚举和实体类型定义为：</span><span class="sxs-lookup"><span data-stu-id="b5f4b-117">For example, consider an enum and entity type defined as:</span></span>

``` csharp
public class Rider
{
    public int Id { get; set; }
    public EquineBeast Mount { get; set; }
}

public enum EquineBeast
{
    Donkey,
    Mule,
    Horse,
    Unicorn
}
```

<span data-ttu-id="b5f4b-118">然后，可以在 `OnModelCreating` 中定义转换，以将枚举值存储为数据库中的字符串（例如，"Donkey"、"Mule" 和 ...）：</span><span class="sxs-lookup"><span data-stu-id="b5f4b-118">Then conversions can be defined in `OnModelCreating` to store the enum values as strings (for example, "Donkey", "Mule", ...) in the database:</span></span>

``` csharp
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder
        .Entity<Rider>()
        .Property(e => e.Mount)
        .HasConversion(
            v => v.ToString(),
            v => (EquineBeast)Enum.Parse(typeof(EquineBeast), v));
}
```

> [!NOTE]  
> <span data-ttu-id="b5f4b-119">`null` 值永远不会传递到值转换器。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-119">A `null` value will never be passed to a value converter.</span></span> <span data-ttu-id="b5f4b-120">这使得转换的实现变得更简单，并使其能够在可以为 null 和不可为 null 的属性之间共享。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-120">This makes the implementation of conversions easier and allows them to be shared amongst nullable and non-nullable properties.</span></span>

## <a name="the-valueconverter-class"></a><span data-ttu-id="b5f4b-121">ValueConverter 类</span><span class="sxs-lookup"><span data-stu-id="b5f4b-121">The ValueConverter class</span></span>

<span data-ttu-id="b5f4b-122">如上所述调用 `HasConversion` 将创建一个 `ValueConverter` 实例，并在属性上对其进行设置。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-122">Calling `HasConversion` as shown above will create a `ValueConverter` instance and set it on the property.</span></span> <span data-ttu-id="b5f4b-123">可以改为显式创建 `ValueConverter`。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-123">The `ValueConverter` can instead be created explicitly.</span></span> <span data-ttu-id="b5f4b-124">例如：</span><span class="sxs-lookup"><span data-stu-id="b5f4b-124">For example:</span></span>

``` csharp
    v => (EquineBeast)Enum.Parse(typeof(EquineBeast), v));

    .Entity<Rider>()
    .Property(e => e.Mount)
    .HasConversion(converter);
```

<span data-ttu-id="b5f4b-125">当多个属性使用相同的转换时，这可能很有用。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-125">This can be useful when multiple properties use the same conversion.</span></span>

> [!NOTE]  
> <span data-ttu-id="b5f4b-126">当前无法在一个位置指定给定类型的每个属性都必须使用相同的值转换器。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-126">There is currently no way to specify in one place that every property of a given type must use the same value converter.</span></span> <span data-ttu-id="b5f4b-127">将来的版本将考虑此功能。</span><span class="sxs-lookup"><span data-stu-id="b5f4b-127">This feature will be considered for a future release.</span></span>
